# static/elementUI 目录说明

本目录为基于 ElementUI 二次封装的通用组件库，旨在统一项目 UI 风格、提升开发效率，并扩展 ElementUI 原生组件的功能。所有组件均遵循统一的开发、命名和国际化规范，便于团队协作和维护。

## 目录结构示例

- **button/**  
  按钮组件，扩展 ElementUI 按钮功能，支持多样化样式与交互。
- **calendar/**  
  日历组件，支持日期选择、区间选择等功能。
- **carousel/**  
  轮播图组件，支持图片/内容轮播展示。
- **chooseArea/**  
  区域选择组件，支持省市区多级联动选择。
- **chooseCity/**  
  城市选择组件，支持城市搜索与选择。
- **chooseDate/**  
  日期选择组件，支持多种日期选择场景。
- **chooseTime/**  
  时间选择组件，支持时分秒选择。
- **datePicker/**  
  日期选择器，封装 ElementUI 日期选择功能，支持多种格式。
- **form/**  
  表单组件，统一表单样式与校验逻辑。
- **input-demo/**  
  输入框组件示例，扩展输入框功能与样式。
- **modalForm/**  
  弹窗表单组件，支持表单与弹窗结合使用。
- **pagination/**  
  分页组件，统一分页交互与样式。
- **pagenation/**  
  另一套分页组件，按需使用。
- **radioGroup/**  
  单选框组组件，支持多样化单选场景。
- **switch/**  
  开关组件，扩展 ElementUI Switch 功能。
- **tabs/**  
  标签页组件，支持多标签切换。
- **tree/**  
  树形控件组件，支持多级数据展示与操作。
- **upload/**  
  文件上传组件，支持多种文件类型与上传场景。

> 说明：如需了解具体组件用法，请参考各子目录下的 README.md 或组件源码。

## 开发规范

- 组件命名需简洁明了，目录结构与功能一一对应。
- 组件需具备良好复用性和可配置性，避免业务强耦合。
- 所有界面可见中文均需国际化，禁止硬编码。
- 组件应有必要的注释和文档，便于团队协作。

## 国际化支持

- 组件内所有界面可见文本均通过 i18nM 方法调用，国际化资源统一管理。
- 新增组件时，需同步补充国际化资源文件。
