# static/js 目录说明

本目录为前端项目的 JS 工具与资源库，涵盖通用方法、API 封装、国际化、主题、存储、mock、路由、指令等，服务于 static 及全局业务模块。

## 目录结构

- **api/**  
  服务器请求路由库，统一管理后端接口路径与请求方法。
- **common/**  
  通用方法库，包含常用工具函数、数据处理等。
- **customDirect/**  
  自定义 Vue 指令库，扩展指令功能，提升开发效率。
- **guard/**  
  页面守卫库，实现页面访问权限控制与路由守卫。
- **http/**  
  HTTP 请求方法封装，统一管理 axios/fetch 等请求逻辑。
- **locales/**  
  国际化语言库，包含 cn.ts、en.ts 等多语言资源文件，所有界面可见文本均通过此目录统一管理。
- **localStorage/**  
  localStorage 操作方法库，封装本地存储相关操作。
- **mock/**  
  mock 数据方法封装，便于本地开发与接口联调。
- **page/**  
  与页面相关的方法库，处理页面级逻辑与操作。
- **router/**  
  vue-router 方法封装，统一路由跳转、参数处理等。
- **socket/**  
  socket 请求封装，管理 WebSocket 相关通信逻辑。
- **store/**  
  pinia 方法封装，统一状态管理相关操作。
- **theme/**  
  主题方法封装，支持多主题切换与样式动态调整。

## 国际化支持

- locales 目录下的 cn.ts、en.ts 等文件为国际化资源，所有界面可见文本均需通过 i18nM 方法调用。
- 资源文件结构与实际界面内容严格对应，注释、类型说明等不做国际化。
- 新增/修改界面文本时，需同步维护多语言资源文件。

## 开发规范

- 工具方法需具备通用性、可复用性，避免与具体业务强耦合。
- 目录结构应清晰、职责单一，便于维护和扩展。
- 命名规范统一，注释清晰，便于团队协作。
